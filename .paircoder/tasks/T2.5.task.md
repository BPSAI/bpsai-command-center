# Objective

Allow operator to resume a completed/failed daemon session from Command Center. Sends resume command to daemon via A2A, daemon runs `claude --resume {session_id}`.

# Implementation Plan

- "Resume" button on session detail (visible for complete/failed sessions)
- Posts resume command to A2A dispatch channel with session_id and operator
- Operator scoping enforced — can only resume own sessions
- UI shows "resume pending" state while daemon picks up command

# Acceptance Criteria

- [ ] "Resume" button on session detail (visible for complete/failed sessions)
- [ ] Posts resume command to A2A dispatch channel with session_id and operator
- [ ] Operator scoping enforced — can only resume own sessions
- [ ] UI shows "resume pending" state while daemon picks up command
- [ ] Error handling: daemon offline, session not resumable
- [ ] Tests: resume command posting, operator scoping, error states

# Verification

- Click Resume on a completed session → resume command posted to A2A
- Daemon picks up command → new session appears with `resumed=true`
- Try to resume another operator's session → blocked
